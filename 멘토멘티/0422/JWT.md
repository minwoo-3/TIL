# JWT

## 쿠키 인증
- 쿠키라는 Key-Value 형식의 문자열 덩어리로 인증하는 방식
- 클라이언트가 웹에 방문할때마다 서버를 통해 브라우저에 작은 기록 정보 파일로 기록됨

### 인증 순서
- 클라이언트가 서버레 요청으 ㄹ보냄
- 서버는 응답할 때 쿠키를 담음
- 이후 클라이언트는 요청을 보낼 때마다 저장된 쿠키를 함께 보냄

## 세션 인증
- 비밀번호 등 클라이언트의 민감한 인증 정보를 서버에 저장하고 관리함
- Setion ID(key에 해당):Value(생성 시간, 마지막 접근 등 User의 속성)
### 인증 순서
- 유저가 로그인을 하면 세션이 서버 메모리상에 저장됨
- 서버에서 브라우저 쿠키에 Session ID저장
- 브라우저는 쿠키를 통해 요청할때마다 Session ID를 쿠키에 담아 전송
- 서버는 블라이언트한테 받은 Session ID와 갖고 있는걸 비교해서 인증수행

## 토큰 인증
- 클라이언트가 서버에 접속하면 인증되었다는 의미로 암호화된 정보가 있는 토큰을 부여
- 클라이언트는 요청을 보낼때마다 토큰을 심어서 보냄

### 인증 순서
- 사용자가 로그인함
- 서버는 사용자에게 유일한 토큰 발급
- 클라이언트는 서버 츨게서 받은 톤클을 쿠키나 스토리지에 저장
- 서버에게 요청할때 토큰을 함께 보냄
- 서버는 토큰을 검증해서 누군지 알아냄
  
## JWT
- 인증에 필로한 정보들을 암호화 한 JSON 토큰
### JWT 구조
- Header.Payload.Signature
 
- Header : 암호화 알고리즘, 토큰 유형
- Payload : 사용될 정보에 대한 내용
- Signature : 비밀 키

### Access Token과 Refresh Token
- Access Token : 유효 기간이 짧은 자원 접근 및 권환 확인용 토큰
- Refresh Token : 비교적 유효 기간이 긴 액세스 토늨 재발급용 토큰

### 인증 순서
- 사용자가 서버에 로그인 인증 요청
- 서버에서 JWT를 쿠키에 담아 클라이언트에 발급
- 클라이언트는 JWT 저장해서 요청을 보낼 때  Access Token을 함께 보냄
- 서버는 클라이언트가 보낸 JWT가 내 서버껀지 확인하여 인증 수행
- 만약 요청에서 엑세스 토큰의 시간이 만료되면 클라이언트는 리프레시 토큰으로 새 엑세스 토큰을 발급 받음
