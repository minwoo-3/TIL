# 객체지향, solid

## 객체 지향
- 프로그래밍에 필요한 데이터를 추상화시켜 상태와 행위를 가진 객체로 만들어 객체들 간 상호작용을 통해 구성하는 방법이다
- java, Python, C++, C#, Kotlin 등이 대표적이다

### 객체 지향의 특징

#### 추상화:
정확하게 표현하기 힘들거나 계속 사용되지만 조금씩 고유기능을 다르게 해줄때 핵심만 정의하여 재사용이 가능하게 한다

#### 상속:
공통된 속성을 하나로 모아놓은 클래스를 만들어서 하위 클래스에 상속시켜 재사용 가능하게 한다

#### 다형성:
하나의 타입으로 여러 객체를 다르게 동작시키는것
- 메서드 오버라이딩 : 부모 클래스의 메서드를 자식 클래스에서 재정의해서 사용하는것
- 메서드 오버로딩 : 메서드 이름은 같이만 매개변수를 다르게 하여 사용하는 것

#### 캡슐화:
데이터와 메서드를 하나의 단위로 묶어 외부에서 접근하지 못하게 보호하는 개념

## 객체 지향의 SOLID원칙

### SRP (Single Responsibility) 단일 책임 원칙:
- 클래스는 한개의 기능(책임)을 가져야 한다
- 클래스를 변경하는 이유는 하나여야한다

### OCP (Open/Closed Principle) 개방-패쇄 원칙:
- 기존 코드를 변경하지 않고 기능을 수정하거나 추가할 수 있도록 설계해야 한다

### LSP (Liskov Substitution) 리스코프 치환 원칙:
- 자식 클래스는 부모 클래스를 대체할 수 있어야 한다
- 하위 클래스가 상위 클래스의 기능을 변경하면 안된다

### ISP (Interface Segregation Principle) 인터페이스 분리 원칙:
- 클라이언트가 사용하지 않는 인터페이스에 의존하지 않도록 해야 한다
- 하나의 큰 인터페이스보다는 여러 개으 작은 인터페이스가 좋다

### DIP (Denpendency Inversion Principle) 의존성 역전 원칙:
- 상위 모듈(추상화)이 하위 모듈(구현)에 의존하는 것이 아니라 반대로 하위 모듈이 상위 모듈에 의존해야 한다
- 의존성을 줄이기 위해 인터페이스를 활용하는 것이 일반적이다
