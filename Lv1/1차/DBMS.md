# DBMS
- DataBase Management System의 약자로 데이터베이스를 조작하는 소프트웨어이다
- 응용 프로그램들이 데이터베이스를 공유하며 사용하는 환경을 제공한다

### DBMS의 기능
- 정의 : 데이터에 대한 형식, 구조, 조건들을 정의하는 기능

- 저장 : 기억장치에 데이터를 저장하는 기능

- 보안 : 하드웨어나 소프트웨어의 오류 또는 권한이 없는 전급으로부터 시스템을 보호하는 기능

- 공유 : 여러 사용자와 프로그램이 데이터베이스에 접근할 수 있도록 공유하는 기능

- 기능 : 데이터의 검색을 위한 질의나 데이터베이스의 갱신. 생성기능

- 유지 : 요구사항의 변화에 따라 반영할 수 있도록 하는 기능

### 대표 DBMS
- Oracle : 
    - 대량의 데이터 처리에 용이
    - Window, Linux, Unix등에서 설치 가능

- MySQL :
    - 현재 오라클에 합병됨
    - Window, Linux, Unix등에서 설치 가능

- MSSQL :
    - 윈도우 특화 

- MariaDB :
    - C++사용
    - MySQL과 동일한 소스 코드 기반

- Tibero :
    - 국내 DBMS
    - SQL과 호환성 좋음

## 관계형 DBMS와 비관계형 DBMS
### 관계형 DBMS(RDBMS)
- 관계형 DB(RDB) : 2차원 테이블 형태로 데이터를 표현한 데이터베이스
- 관계형 DBMS(RDBMS) : 관계형 DB를 생성하고 수정하고 관리할 수 있는 소프트웨어
    - 모든 데이터를 2차원 테이브를 표현
    - 테이블은 row와 column으로 이루어진 기본 데이터 저장 단위
    - 확장에 용이

### 비관계형 DBMS
- 비관계형 DB : 고정된 테이블 스키마 없이 키-값, 그래프 등 유연한 데이터 구조를 사용
- 비관계형 DBMS : 비관계형 DB를 관리하는 DBMS
    - MongoDB가 가장 대중적
    - 단순 검색과 같이 복잡하지 않은 데이터 저장에 용이
