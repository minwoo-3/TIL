## 객체지향 프로그래밍
* 프로그래밍에서 필요한 데이터를 추상화 시켜 행동과 상태를 가진 객체로 만들고, 객체들 간의 상호작용을 통해 로직(논리적인 구조)을 구성하는 프로그래밍 방법이다
* 객체 지향은 다음 4가지 특징이 있다
### 추상화
* 알고 있지만, 표현하기 힘든것들을 중요한 부분이나 특징만 잡아 설명하는것
* 프로그래밍에선 클래스를 정의 할 때 불필요한 부분은 생략하는 것이다
* 아이폰과 삼성폰을 스마트폰으로 묶을 수 있는것이 예이다

### 상속
* 여러 개체들이 지닌 공통된 특성을 부각시켜 개념이나 법칙으로 성립하는 과정이다
* 삼성폰,휴대폰,통신기기,전자제품처럼 중복된 속성들이 있는것이 있다면 그 공통 속성을 하나의 클래스로 만들어 상속 하는 것이 예이다

### 다형성
* 같은 자료형에 여러 가지 타입의 데이터를 대입하여 다양한 결과를 얻어낼 수 있는 성질을 의미한다
* 대표적인 예
    * 메서드 오버라이딩
        * 부모 클래스의 메서드를 자식 클래서에서 재정의해서 사용하는 것
    * 메서드 오버로딩
        * 같은 이름의 메서드를 매개변수와 타입과 개수로 오버로딩하여 사용하는 것

### 캡슐화
* 데이터와 메서드를 하나의 단위로 묶어, 외부에서 접근하지 못하도록 보호하는것
* 외부에서 알 필요 없는 부분을 감춘다
