# Spring이란?
- 자바로 웹 서비스나 서버를 만들 때 사용하는 프레임워크
- 웹 서비스나 서버를 만드는걸 도와주는 틀
## Spring의 4원칙
### POJO 프로그래밍을 지향
- POJO:Plan Old Java Object의 약자로 순수 Java만을 통해서 생성한 객체
- 외부 라이브러리나 모듈을 사용하지 않고 순수 Java만들 사용한다
- 순수 Java만을 이용하기 때문에 외부 기술이나 규약의 변화에 얽매이지 않아, 보다 유연하게 변화와 확장에 대처 가능
- 객체지향 설계를 제한없이 적용할 수 있어 코드가 단순해지므로 디버깅도 쉬움
### IoC/DI
#### IoC : 
제어의 역전이라고 하며 객체의 생성과 실행 흐름을 프레임워크가 자동으로 하는것이다
#### DI : 
의존성 주입이라고 하며 객체를 직접 만들지 않고, 외부에서 넣어주는 방식
#### IoC/DI
- 객체를 직접 만들지 않고 스프링이 대신 관리해줌
- DI가 객체를 외부에서 넣으면 IoC를 통해 제어권이 프레임워크로 넘어간다

### AOP(관심 지향 프로그래밍)
#### 핵심 관심 사항 :
- 앱의 핵심 기닝과 관련된 관심 사항
- 예 : 커피 주문 앱의 메뉴 등록, 주문, 주문 병경 등
#### 공통 관심 사항 :
 - 모든 핵심 관심 사항에 공동적으로 적용되는 관심 사항
 - 예 : 위 주문, 주문 병경, 메뉴등록등에 모든 핵심 관심 사항에 적용되는 로깅이나 보안 등과 관련되 기능
#### AOP : 
공통 관심 사항과 관련된 기능들을 별도의 객체로 분리하여 객체의 메서드를 통해 공동 관심 사항을 구현하는 것

### PSA(일관된 서비스 추상화)
#### JDBD :
스프링이 Java를 사용하여 DB에 접근하는 방법을 규정한 인터페이스를 말한다
#### PSA :
특정 기술과 관련된 서비스를 추상화하여 일관된 방식으로 사용될 수 있도록 한 것

## Spring Boot
- 스프링을 쉽고 빠르게 사용할 수 있게 만든 도구

#### 스프링의 단점 보완 :
스프링의 단점을 자동으로 처리해준다
- 설정이 많음
- 서버 따로 설치해야함
- 시작하기 어려움

### Spring Boot의 특징
- 설정 자동
    - DB 연결
    - 웹 설정
    - 라이브러리 설정
- 내장 서버
    - Tomcat, Jetty 등이 포함됨
- 최소 코드로 서버 실행 가능
- 필요한 기능을 한번에 추가 가능함

### 스프링과 비교
- 스프링
    - 재료부터 레시피까지 다 직접 준비
- 스프링 부트
    - 밀키트
