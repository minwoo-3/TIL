# RESTful API란
- RESTful API : 두 컴퓨터 시스템이 인터넷을 통해 정보를 안전하게 교환하기 위해 사용하는 인터페이스

### API란?
- 어플리케이션 프로그래밍 인터페이스의 약자이며 다른 소프트웨어 시스템과 통신하기 위해 따라야 하는 규칙을 정의한다
- 다른 앱이 프로그래밍 방식으로 앱 통신을 할 수 있도록 표시하거나 생성
    - 웹 API는 클라이턴으롸 웹 리소스 사이의 게이트웨이 정도이다
- #### 클라이언트
    - 웹에서 정보를 사용하려는 사용자이다
    - 사람이거나 소프트웨어 시스템일 수 있다
- #### 리소스
    - 다양한 앱이 클라이언트에 제공하는 정보
    - 모든 정보가 포함된다
### REST란?
- API 작동 방식에 대한 조건을 부과하는 설계도 같은 것이다
- REST아키텍쳐를 구현하는 웹 서비스를 RESTful이라고 한다
- #### 자원(Resource)
    - 모든 자원은 고유한 URL로 구별된다
- #### 행위(Verb)
    - 자원에 대한 작업은 HTTP 메서드로 한다
- #### 표현(Representation)
    - 클라이언트와 서버가 데이터를 교환할때 JSON, XML 등의 형식을 사용한다
- #### 무상태성(Stateless)
    - 서버는 클라이언트의 상태를 저장하지 않아, 요청마다 필요한 정보를 모두 포함해야한다
