# HTTP
## HTTP란?
HTTP란 HyperText Transfer Protocol의 약자로<br> HTML 문서를공유하기 위한 약속(규칙)이다
## HTTP와 HTTS의 차이점
HTTPS는 HTTP에 TLS라는 암호화 프로토콜을 통해<br> 안정성을 확보한 것이다
### HTTPS 작동 방식
* 사용자가 URL을 주소창에 적는다
* 브라우저는 서버에 인증서를 요구한다
* 서버는 퍼블릭키가 포함된 인증서를 반환한다
* 브라우저에서 인증이 되면 브라우저는 비밀 세션 키를 암호화해서 서버에 보낸다
* 서버는 세션 키를 프라이빗 키로 해독해서 세션 키를 찾고 브라우저에 승인 메세지를 보낸다
* 서로 같은 세션 키로 안전하게 메세지를 주고 받는다

HTTP는 권한이 없는 사람도 볼 수 있어<br> 신뢰성이 부족하기에 HTTPS를 많이 쓴다
## HTTP Mothod
HTTP Mothod란 서버가 수행해야 할 동작을 지정하여 요청하는 방법이다<br>
HTTP의 종류는 다음과 같다

### HTTP Method 종류
* GET : 리소스(자원) 조회

* POST : 데이터 추가, 등록
* PUT : 리소스 대체, 수정 혹은 생성
* DELETE : 리소스 삭제
* PATCH : 리소스 부분 병경
* HEAD : GET과 동일(body 부분 제외)
* OPTION : 통신 옵션 확인
* CONNECT : 자원으로 식별되는 서버에 연결 요청

## HTTP 상태코드
HTTP 응답 상태 코드는 특정<br>HTTP 요청이 성공적으로 완료되었는지 <br>알려주는 메세지 같은것이다<br>
주요 상태코드 종류와<br>주요 상태코드는 다음과 같다
### 상태코드 종류
* 1XX : 정보제공

* 2XX : 성공
    * 200 : 성공
    * 201 : 생성됨
    * 202 : 허용됨

* 3XX : 리다이렉션(필요한 추가 동작)
    * 301 : 영구 이동
    * 303 : 다른 위치 보기
    * 307 : 임시 리다이렉션 요청 필요

* 4XX : 클라이언트 에러
    * 400 : 잘못된 요청
    * 401 : 권한 없음
    * 403 : 금지됨
    * 404 : 찾을 수 없음

* 5XX : 서버 에러
    * 500 : 내부 서버 오류
    * 501 : 구현되지 않음
    * 502 : 불량 게이트웨이

## HTTP 무상태성
무상태성(stateless)이란<br>
서버가 클라이언트의 상태를<br>
모르고 있는(저장하지 않는) 상태이다

### 장점
* 서버에서 관리해야하는 부담이 줄어든다
* 클라이언트와 연결을 유지 할 필요X
* 서버 대기 시간이 줄어든다
### 한계
* 이전 상태에 대한 설명을 다시 해야한다
* 쿠키, 서버 세션, 토큰 등이 필요하다
