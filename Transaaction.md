# 트랜잭션

### 트랜잭션(Trasaction)이란?
- DB의 완전성을 보장하기 위한 것
- 상태를 변화 시키기 위해 수행하는 하나의 논리적인 작업 단위
- 작업을 하나의 묶으로 처리해서 전부 선공하거나 전부 실패하게 만든다

### 트랜잭션의 핵심 특징 (ACID)
#### 원자성(Atomicity)
- 트랜잭션이 DB에 모두 반영되던가 전혀 반영되지 않아야한다
#### 일관성(Consistency)
- 트랜잭션이 진행되는 동안 DB가 변경되더라도 업데이트된 DB로 진행하는 것이 아닌 처음 참조하나 데이터들로 진행
#### 독립성(Isolation)
- 둘 이상의 트랜잭션이 동시에 실행되고 있을때 어떠한 하나의 트랜잭션이 다른 트랜잭션에 끼어들 수 없음
#### 지속성(Durability)
- 트랜잭션이 완료되었을 경우 결과는 영구적으로 반영

### Comit / Rollback
#### Comit :
하나의 트랜잭션이 성공적으로 수행되었으며 데이터베이스가 일관성 있는 상태일 때를 말한다
#### Rollback :
하나의 트랜잭션이 비정상적으로 종료되어 트랜잭연이 깨진경우 처음부터 다시 시작하거나 부분적으로 결과를 다시 취소 시킴을 의미한다
